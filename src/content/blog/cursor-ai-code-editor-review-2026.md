## Cursor 是什麼？它和 VS Code 有什麼不同？

Cursor 是一款以 VS Code 為基礎打造的 AI 程式編輯器。如果你已經用 VS Code，切換過來幾乎沒有學習成本——快捷鍵、外掛、介面都高度相容。

但差異在核心設計理念上：

VS Code 是「工具 + 你」，AI 是附加的外掛。  
Cursor 是「你 + AI 一起工作」，AI 從一開始就深度整合在編輯器裡。

具體來說，Cursor 的主要功能包含：

- **Tab 自動補全**：不只補單行，可以根據上下文預測你接下來要做什麼，提前補好整個函式或邏輯區塊
- **Cmd+K（行內編輯）**：選取一段程式碼，直接用自然語言告訴 AI 你要改什麼，它直接在原地修改
- **Cmd+L（Chat）**：開啟側邊欄與 AI 對話，AI 可以讀取你整個專案的上下文，不只是單一檔案
- **Codebase 理解**：Cursor 可以索引你的整個 repo，讓 AI 回答「這個函式在哪裡被呼叫過」這類問題

## 哪種開發者最適合 Cursor？

根據 Cursor 的產品定位和開發者社群的反饋，以下幾種情境最能發揮它的價值：

**1. 需要頻繁切換語言或框架的工程師**  
如果你同時維護 Python 後端、TypeScript 前端，還偶爾要改 CI/CD 的 YAML，記住每個語言的細節語法會消耗大量認知資源。Cursor 可以在你不確定的時候直接補全，減少查文件的頻率。

**2. 在既有 codebase 工作的開發者**  
加入新專案或接手舊程式碼時，理解整個架構需要時間。Cursor 的 codebase indexing 讓你可以直接問「這個 API endpoint 是在哪裡定義的」，而不是盲目搜尋。

**3. 獨立開發者或小型團隊**  
沒有資深同事可以問、沒有 code review 文化，AI 的角色就更像是隨時在線的協作者。

## 它的限制是什麼？

誠實說，Cursor 不是萬能的。

**AI 補全不一定正確**：它會根據上下文猜測，但猜錯的情況不少，尤其是複雜的商業邏輯或高度客製化的架構。你仍然需要理解程式碼，而不是盲目接受它的建議。

**免費版有限制**：免費版包含有限的 AI 請求數量，密集使用的開發者很快會碰到上限。Pro 版每月約 20 美元。

**對初學者可能是雙面刃**：如果你還在學習基礎語法，過度依賴 AI 補全可能讓你跳過了真正理解程式碼的過程。

## 和 GitHub Copilot 比，選哪個？

這是很多開發者會問的問題。簡單的差異：

- **GitHub Copilot**：外掛形式，可以裝在 VS Code、JetBrains 等各種編輯器，靈活度高，與 GitHub 生態深度整合
- **Cursor**：完整編輯器，AI 整合更深，chat 功能更強，對整個 codebase 的理解更完整

如果你已經很依賴特定編輯器的生態系統，Copilot 可能更順。如果你願意換一個編輯器換取更深度的 AI 協作體驗，Cursor 值得試。

兩者都有免費版可以先試用，不需要馬上做決定。
